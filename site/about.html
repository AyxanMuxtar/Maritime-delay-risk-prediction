How this works

The problem

Maritime operators in the Caspian Sea face weather-driven delays from storms, fog, heavy precipitation, and dangerous waves. This site forecasts the probability that any given day in the upcoming month will be operationally disrupted at five major ports: Baku (Azerbaijan), Aktau (Kazakhstan), Anzali (Iran), Turkmenbashi (Turkmenistan), and Makhachkala (Russia).

What counts as a risk day

A day is flagged as risky if any of eight thresholds are breached: storm-force sustained wind, dangerous gusts, heavy precipitation, snowfall, low visibility (fog), or wave heights derived from wind via the SMB formula. Thresholds are calibrated separately per city to account for local climatology.

Models

An XGBoost gradient-boosted classifier predicts the daily binary risk label given weather features. The model is trained on ten years of ERA5 reanalysis data (2015–2024) for each port. Input features were curated through a structured Day-7 hypothesis-testing and feature-selection process: pairwise correlation, variance inflation factor analysis, and automatic dropping of redundant features at |r| > 0.85.

For days within the 16-day Open-Meteo forecast horizon, the model uses real forecast features as input. For days beyond that horizon, predictions fall back to a per-(city, day-of-year) climatology lookup, smoothed with a ±7-day rolling window. Each daily prediction is tagged with its source so users can distinguish high-confidence model predictions from lower-confidence climatological estimates.

Pipeline

The full pipeline runs automatically on the 1st of each month via a GitHub Actions cron job. It fetches new historical data through Open-Meteo, validates it against seven automated quality gates, retrains the model on the latest dataset, generates predictions for the upcoming month, and commits the resulting CSVs back to the source repository. This site auto-deploys whenever the repo updates, so the forecasts you see are always current.

Honest limitations

Wave height is computed from wind via a fetch-limited SMB formula, not measured directly — the actual marine API for the Caspian wasn't usable. Visibility data for years 2015–2021 uses median imputation since the Open-Meteo Historical Forecast API only covers 2022 onward. Bootstrap confidence intervals on the model's metrics may be slightly optimistic because daily weather is temporally autocorrelated. The 10-year training window is short for detecting climate-scale trends.

Source code

All code, notebooks, and methodology details are open source on GitHub. The codebase includes 8 day-by-day Jupyter notebooks documenting the entire build process from API exploration to deployment, plus a production Python pipeline with quality gates, logging, and audit tables.
